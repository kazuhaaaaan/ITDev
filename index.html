Analisis Kapasitas Aktual

Infrastruktur saat ini mengandalkan layanan Home Broadband 500 Mbps. Metrik aktual menunjukkan penurunan performa hingga batas ~200 Mbps akibat limitasi Shared Broadband.

Identifikasi Masalah: Contention Ratio

Ketiadaan FUP (Fair Usage Policy) tidak ekuivalen dengan ketersediaan Bandwidth Dedikasi. Jaringan saat ini mengalami isu Contention Ratio, memperebutkan kapasitas di level distribusi ISP dengan entitas residensial lainnya.

Proyeksi Beban Perangkat (Skalabilitas)

Beban saat ini (Lantai 1) adalah 48 perangkat aktif (rasio ~4 Mbps/perangkat saat utilisasi puncak). Aktivasi area Lantai 2 akan meningkatkan total beban menjadi ~76 perangkat. Hal ini memicu penurunan bandwidth hingga ~2.6 Mbps/perangkat—jauh di bawah standar operasional untuk kebutuhan paralel Video Conference dan transfer aset visual berskala besar.

Arsitektur Topologi Jaringan

Desain infrastruktur dioptimalkan menggunakan pendekatan sentralisasi Layer 3 Routing dan distribusi Layer 2 Switching untuk mitigasi latensi.

Alur Distribusi Perangkat

  • 🧠 Core Gateway (MikroTik)

    Pusat kontrol utama. Menangani Inter-VLAN Routing, manajemen bandwidth (PCQ), dan orkestrasi WiFi terpusat via CAPsMAN.

  • 🔀 Core Switch (Managed L2)

    Agregator utama. Menerima trunk VLAN dari Gateway dan mendistribusikan jalur ke Switch area spesifik secara terstruktur.

  • 🏢 Switch Distribusi

    Fasilitator *wired connection* per departemen (terutama PC Editing) untuk memastikan stabilitas transfer data lokal tanpa utilisasi WiFi.

  • 📡 Access Point (WiFi)

    Distribusi nirkabel terintegrasi dengan kapabilitas seamless roaming, menjaga koneksi rapat virtual tetap stabil saat mobilitas antar area.

Justifikasi Pengadaan

Proporsional Area

Rasio 5 unit AP dan 4 Switch Distribusi dihitung secara matematis untuk eliminasi blind spot di Lantai 1.

Zero Bottleneck Processor

Router RB5009 (ARM 64-bit) diproyeksikan untuk mengelola beban puncak inter-VLAN routing bagi ~76 perangkat secara bersamaan.

Skalabilitas (Future-Proof)

Desain siap ekspansi. Penambahan area Lantai 2 hanya memerlukan tarikan kabel trunk tunggal tanpa merestrukturisasi Core Router.

Efisiensi Finansial

Bukan perangkat *consumer-grade* yang rentan hang, namun tidak perlu investasi level *Enterprise* yang berlebih (High ROI).

Executive Summary: Urgensi Implementasi

1. Limitasi Kapasitas Hardware Lama

Router *consumer-grade* memiliki batas optimal utilisasi ~25 perangkat. Memaksakan beban 76 perangkat akan memicu degradasi performa masif.

2. Mitigasi Latensi Internal

Distribusi kabel (Switch) untuk stasiun kerja berat dipadukan dengan *Seamless Roaming* AP mengeliminasi isu putusnya koneksi.

3. Biaya Downtime vs Infrastruktur

Kerugian finansial akibat tertundanya produktivitas secara signifikan melampaui biaya investasi alat jaringan (*CAPEX*).

4. Simbiosis Alat dan ISP

Meningkatkan paket ISP tanpa upgrade hardware menciptakan *bottleneck*. Keduanya harus diselaraskan untuk performa maksimal.

Kesimpulan

Pembaruan topologi dan transisi layanan ISP ini diposisikan sebagai Inisiatif Business Continuity. Hal ini krusial untuk mengamankan arus operasional digital dan memastikan produktivitas tim dapat berjalan pada tingkat efisiensi tertinggi tanpa interupsi.
